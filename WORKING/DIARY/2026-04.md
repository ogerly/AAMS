### 2026-04-09 — SCIENCE verworfen → RFL + Naming Schema eingeführt

**Kontext:** Knowledge Validation Layer (SCIENCE) als fünfter Dokumentations-Layer konzipiert und dann verworfen. Zu komplex für eine Spec. Bricht local-first-Prinzip. SCIENCE wandert als Feature ins Agent-Loop-Framework.

**Stattdessen:** RFL (Reflection Protocol Step) — ein Konsistenzprüfungs-Schritt im Session-Start-Contract. Löst das häufigste Failure-Pattern: Agent widerspricht sich über Sessions hinweg. 3-Stufen-Fallback: Pattern-Match → LTM → chronologisch.

**Naming Schema** eingeführt: `WP-{DATE}-{TOPIC}-{SUBTOPIC}-{description}.md` für Workpapers, `WH-{NNN}-{TOPIC}-{description}.md` für Whitepapers. TOPIC-Tags als primärer Ordnungsschlüssel. Hybrid-Ansatz: Spec empfiehlt, Framework erzwingt.

**Kernerkenntnis:** Spec = Körper (Struktur, Speicher, Protokoll). Framework = Geist (Reasoning, Recherche, Validierung). Vier Layer bleiben — RFL ist Protokollschritt, nicht Layer.

**Issues:** #36 (Naming Schema), #37 (RFL), #38 (WP-001), #39 (v1.2.0 Release)
